# templates-02

## Objective

Reverse-engineer a set of document templates observed from another team's workflow. Starting from file names only, scaffold plausible content using best judgment, then analyze where these templates fit within the product lifecycle and how they relate to the `templates-01` library and the Software Lifecycles hierarchy.

## Process

1. **Observe** — identify file names spotted in another team's presentation
2. **Scaffold** — build out each file with inferred content based on the file name and context
3. **Reflect** — determine what problem this template set is solving and who it's for
4. **Map** — locate these templates within the lifecycle framework (`Software-Lifecycles-Overview.md`) and compare to `templates-01`
